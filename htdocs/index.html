cvsfs4hurd, v. 0.1

what is it?

cvsfs4hurd is a translator for the GNU Hurd, enabling to bind a (remotely located) cvs module right into your local filesystem. It allows you to view version controlled files right the same way, you look at regular, locally stored, ones.

cvsfs4hurd

cvsfs4hurd is actually the first translator I've ever written. It's actually one step towards the GNU Hurd for me (I haven't done much with it before starting to write the translator). cvsfs4hurd is not the very stable thing, lacking some more and closer testing as well (which I will definitely do if there is some more time on my side).

see here for more information on cvsfs4hurd (and download possibility).

fuse4hurd

Short time after publishing my cvsfs4hurd in October 2004 I got more and more in touch with the cvsfs project for Linux (http://cvsfs.sourceforge.net) by Petric Frank.

Since Petric wanted to rewrite his cvsfs (mainly it's kernel part) I convinced him that using fuse (a library to introduce userspace filesystems to Linux) is a good idea. In turn I decided to write a interface between Hurd's netfs library and fuse, which I call fuse4hurd - mainly to have his cvsfs available on the Hurd (which still has some advantages to my cvsfs implementation).

fuse4hurd is IMHO not ready to be released, however you might want to look into this projects CVS to find out some more about it :)
