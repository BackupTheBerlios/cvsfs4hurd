cvsfs4hurd, v. 0.1

what is it?

cvsfs4hurd is a translator for the GNU Hurd, enabling to bind a (remotely located) cvs module right into your local filesystem. It allows you to view version controlled files right the same way, you look at regular, locally stored, ones.

Current release

The latest version is 0.1. This is the first release at all, therefore please don't expect an bug-free set of programs. If you suffer from a bug, please, please tell me.

If you're used to using translators on the Hurd you probably won't have trouble using it. As a first little intro you may want to read this webpage, the README file and the text cvsfs spits out, when you do this: 

    $ ./cvsfs --help

In case you want some more documentation, you may want to look at the available man page or the texinfo documentation.

Source packages

The main release is C source code, which can be installed and run on GNU Hurd systems. Get copies from berlios.de's file release system: here.

CVS access

To keep up with the development of cvsfs4hurd you can either browse latest CVS with viewcvs, get a checkout or (if you already have cvsfs installed) just mount it.

Bug reports

cvsfs4hurd is currently under heavy development and is far from a mature state. Therefore, if you installed a packaged version, you probably want to consider installing most recent CVS version first. If the bug is in there as well, you (might) want to either file a bug report or send me an email to ssiegl@gmx.de.

First steps

If you've successfully compiled and installed cvsfs you probably want to try it out. To do so, just try mounting the cvs module of xmltVR3 into your local filesystem tree:

  1. create a new directory acting as the underlying node of choice (aka mount-point) 
        $ mkdir -p cvsfs_testing
  2. start the translator 
        $ settrans -a cvsfs_testing /sbin/cvsfs cvs.sourceforge.net /cvsroot/xmltvr3 xmltVR3
  3. now use the files below the cvsfs_testing/ directory as you're used to, e.g. using emacs, ls, or what ever ...
  4. to get rid of the included filesystem, just leave the cvsfs_testing/ directory and enter 
        $ settrans -g cvsfs_testing
